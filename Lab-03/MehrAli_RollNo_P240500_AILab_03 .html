Problem Statement¶

The map of Romania is represented as a graph, with cities as nodes and roads as edges. We are given a starting city and a goal city. We need to write two functions, one using BFS and one using DFS, to find a path between the starting city and the goal city.

  • G: A NetworkX graph object representing the map of Romania
  • start: The starting city
  • goal: The goal city

The functions should return a list of cities from start to goal, or None if no path exists.

Step 4: BFS Function¶

BFS uses a queue. It explores cities level by level (nearest cities first). It always finds the path with the fewest stops.

In [4]:
def bfs(G, start, goal):
    queue = [[start]]
    visited = [start]

    while queue:
        path = queue.pop(0)
        city = path[-1]

        for next_city in G.neighbors(city):
            if next_city not in visited:
                new_path = path + [next_city]
                if next_city == goal:
                    return new_path
                visited.append(next_city)
                queue.append(new_path)

    return None

Step 5: DFS Function¶

DFS uses a stack. It explores as deep as possible into one path before backtracking. It finds a valid path but not necessarily the shortest one.

In [5]:
def dfs(G, start, goal):
    stack = [[start]]
    visited = [start]

    while stack:
        path = stack.pop()
        city = path[-1]

        for next_city in G.neighbors(city):
            if next_city not in visited:
                new_path = path + [next_city]
                if next_city == goal:
                    return new_path
                visited.append(next_city)
                stack.append(new_path)

    return None

bfs_path = bfs(G, 'Arad', 'Bucharest')

Short Explanation¶

BFS DFS
Data Structure Queue — pop(0) Stack — pop()
Explores Level by level Deep first
Shortest path? Yes Not always
Returns List of cities or None List of cities or None

The only real difference between BFS and DFS in the code is pop(0) vs pop(). Everything else is the same.
